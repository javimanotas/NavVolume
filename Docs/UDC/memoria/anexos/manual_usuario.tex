\chapter{Manual de usuario}
\label{anexo:usuario}

Este apéndice describe el funcionamiento y uso de la herramienta desde el editor de Unity.
Está orientado tanto a perfiles técnicos como a diseñadores de niveles y artistas, buscando que su configuración sea lo más intuitiva posible.
Para consultar el proceso previo de instalación del paquete, véase el Apéndice~\ref{anexo:instalacion}.

\section{Agentes}

El primer paso para utilizar la herramienta consiste en definir la configuración base para un tipo de agente.
Para ello, se hace clic derecho sobre la ventana de proyecto y se selecciona la opción \texttt{Create > NavVolume > Agent Type}, tal y como se ilustra en la Figura~\ref{fig:creacionagente}.

\begin{figure}[htp]
  \centering
  \includegraphics[width=0.75\textwidth]{imaxes/creacionagente.png}
  \caption[Creación de la configuración para un agente.]
  {Creación de la configuración para un agente.
  Imagen obtenida por medio de una captura de pantalla.}
  \label{fig:creacionagente}
\end{figure}

Una vez creado este archivo, al seleccionarlo en el inspector podremos modificar los siguientes parámetros principales (visibles en la Figura~\ref{fig:configuracionagente}):

\begin{itemize}
    \item \textbf{Radius:} Radio físico del agente, empleado para evitar colisiones tanto con la geometría del escenario como con otros agentes.
    \item \textbf{Cost mode:} Determina cómo calcula el coste de las rutas.
    Si se elige \textbf{Node Count}, el algoritmo trata a todos los nodos por igual.
    Esto calcula las rutas más rápido y hace que los agentes tiendan a moverse por los espacios más abiertos.
    Si se elige \textbf{Euclidean distance}, calcula la distancia real, dando resultados más exactos pero consumiendo más rendimiento.
    En espacios muy amplios, se recomienda usar el primer método, ya que el algoritmo de suavizado final suele compensar esa pérdida de precisión.
    \item \textbf{Heuristic Weight:} El peso que se le da a la heurística durante la búsqueda.
    Un valor de 1 garantiza siempre el camino óptimo.
    Valores mayores calcularán la ruta más rápido, sacrificando un poco la precisión del camino resultante.
    \item \textbf{Avoidance Neighbor Range:} Distancia máxima a la que se tienen en cuenta a otros agentes para intentar esquivarlos.
    \item \textbf{Avoidance Max Neighbors:} Número máximo de agentes que se procesan al mismo tiempo para la evasión.
    \item \textbf{Avoidance Time Horizon Agents:} Cuántos segundos mira el algoritmo hacia el futuro para predecir y evitar colisiones con otros agentes.
    \item \textbf{Avoidance Time Horizon Obstacles:} Cuántos segundos mira hacia el futuro para evitar colisiones con obstáculos dinámicos.
\end{itemize}

  \centering
  \includegraphics[width=0.75\textwidth]{imaxes/configuracionagente.png}
  \caption[Parámetros de configuración para un agente.]
  {Parámetros de configuración para un agente.
  Imagen obtenida por medio de una captura de pantalla.}
  \label{fig:configuracionagente}

Para conseguir que un objeto (\texttt{GameObject}) se mueva de forma autónoma, debemos añadirle el componente \texttt{NavVolumeAgent} y asignarle el archivo de configuración que acabamos de crear.
Desde su inspector (Figura~\ref{fig:componenteagente}) podemos modificar parámetros como:

\begin{itemize}
\item \textbf{Agent Type:} El fichero de configuración del agente descrito anteriormente.
    \item \textbf{Speed:} La velocidad máxima a la que se desplazará el agente.
    \item \textbf{Angular Speed:} La velocidad a la que rotará el agente, medida en grados por segundo.
    \item \textbf{Is Avoidance Enabled:} Una casilla para activar o desactivar la esquiva automática de obstáculos y de otros personajes.
    \item \textbf{Freeze Rotation:} Por defecto, el agente rota para mirar hacia donde se mueve.
    Al marcar estas casillas, podemos bloquear su rotación en ejes específicos.
    Por ejemplo, si nuestro agente es un dron, podríamos interesarnos en que solo pueda rotar sobre el eje Y.
\end{itemize}

\section{Volumen de navegación}

Para que los agentes puedan moverse, necesitan que exista un volumen espacial transitable.
Para crearlo, generamos un objeto vacío en la escena y le añadimos el componente \texttt{NavVolumeSpace}, tal y como se muestra en la Figura~\ref{fig:componentevolumen}.

\begin{figure}[htp]
  \centering
  \includegraphics[width=0.75\textwidth]{imaxes/componentevolumen.png}
  \caption[Parámetros del componente \texttt{NavVolumeSpace}.]
  {Parámetros del componente \texttt{NavVolumeSpace}.
  Imagen obtenida por medio de una captura de pantalla.}
  \label{fig:componentevolumen}
\end{figure}

El primer paso es asignarle un fichero de configuración de agente.
Cada volumen solo permite un tipo de agente, ya que la forma en la que se construye el espacio navegable depende del tamaño de este.
A su vez, a los agentes que estén dentro del volumen se les asignará automáticamente dicho espacio.
Si se solapan varios volúmenes del mismo tipo, el agente escogerá aquel que tenga configurada una mayor prioridad.

Otros parámetros ajustables incluyen el tamaño total del volumen (el lado del cubo contenedor), el número de capas de subdivisión espacial y la máscara de colisión, que determina qué elementos físicos del entorno actuarán como paredes u obstáculos.


Para facilitar el diseño del nivel, el componente incluye unos manejadores (\textit{handles}) en la vista de escena, visibles en la Figura~\ref{fig:manejadores}.
Al hacer clic y arrastrarlos, el tamaño del volumen se ajusta directamente de forma visual e intuitiva.

\subsection{Construcción de la estructura}

Para generar la estructura de navegación interna, el sistema ofrece tres modos distintos:

\begin{itemize}
    \item \textbf{Baked:} El volumen se calcula mientras editamos el juego y se guarda en un archivo en el disco duro.
    Al iniciar la partida, simplemente se carga este archivo, logrando el mejor rendimiento posible.
    Para usarlo, primero debemos crear un archivo haciendo clic derecho en la ventana de proyecto (\texttt{Create > NavVolume > Baked Data}), asignarlo al componente y pulsar el botón \textit{Bake}.
    \item \textbf{Build On Awake:} El volumen se calcula automáticamente justo al arrancar la escena.
    Es muy útil si el juego utiliza sistemas de generación procedural donde el escenario cambia y no se puede precalcular.
    \item \textbf{Manual:} El volumen solo se construye cuando lo ordenamos llamando a la función \texttt{Build()} mediante código.
\end{itemize}

Una vez construido, el sistema muestra una serie de estadísticas (Figura~\ref{fig:bakestats}), como el número de nodos por capa o el ahorro de memoria conseguido frente a una representación tradicional de cuadrícula.
Adicionalmente, se detalla cuánto tiempo ha durado cada fase de la construcción, de mayor a menor.
Estos datos de tiempo no se guardan en el archivo y varían entre sesiones, ya que dependen directamente de la potencia del ordenador en el que se ejecuta el proceso.

\section{Obstáculos dinámicos}

La herramienta permite incluir elementos que obstaculicen el paso y que puedan moverse durante la partida.
Para lograrlo, hay que añadir el componente \texttt{NavVolumeObstacle} al objeto deseado.
Desde el inspector de este objeto (Figura~\ref{fig:obstaculodinamico}), podemos configurar:

\begin{itemize}
    \item \textbf{Shape:} La forma del obstáculo (puede ser un cubo o una esfera).
    \item \textbf{Center:} El centro de la figura con respecto a la posición original del objeto.
    \item \textbf{Size:} Las dimensiones (ancho, alto y largo) si es un cubo, o un único valor decimal (el radio) si es una esfera.
\end{itemize}

\begin{figure}[htp]
  \centering
  \includegraphics[width=0.75\textwidth]{imaxes/obstaculodinamico.png}
  \caption[Parámetros del componente \texttt{NavVolumeObstacle}.]
  {Parámetros del componente \texttt{NavVolumeObstacle}.
  Imagen obtenida por medio de una captura de pantalla.}
  \label{fig:obstaculodinamico}
\end{figure}

No es obligatorio que estos objetos tengan componentes de colisión físicos (\textit{Colliders} de Unity) para que el sistema los detecte.
Sin embargo, si los tienen, es crucial asegurarse de que pertenezcan a una capa (\textit{Layer}) que el volumen de navegación esté ignorando en su máscara de colisión inicial.
De lo contrario, el volumen pensará que la posición original del objeto es un muro permanente y nunca permitirá navegar por ahí, aunque el objeto se mueva después.

\section{Elementos de \textit{demo}}

Para facilitar las primeras pruebas a los nuevos usuarios, se ha incluido un \textit{script} de utilidad llamado \texttt{FlyToTarget}.
Este componente permite que los agentes se muevan hacia un destino sin necesidad de programar nada.

Para usarlo, basta con añadirlo a nuestro agente y asignarle un objetivo en el campo \texttt{Target}, como se ilustra en la Figura~\ref{fig:flytotarget}.
El agente se dirigirá automáticamente hacia él, y si el objetivo cambia de posición, la ruta se recalculará sola.
Al ser un elemento de demostración y no parte del núcleo del sistema, no tiene un icono personalizado a diferencia del resto de componentes de la herramienta.

\section{Uso desde código}

\subsection{\texttt{NavVolumeSpace}}

\begin{itemize}
    \item \texttt{Bounds VolumeBounds \{ get; \}} Debuelve la caja de contorno (\textit{bounding box}) alineada con los ejes que delimita el volumen en coordenadas de mundo, centrada en la posición del \texttt{transform} y con el lado igual al tamaño configurado. Está disponible incluso antes de construir el volumen.
    \item \texttt{bool IsReady \{ get; \}} Indica si los datos de navegación ya se han construido o cargado y, por tanto, el volumen puede responder consultas y peticiones de caminos. Mientras sea \texttt{false}, los métodos de consulta devuelven valores seguros por defecto en lugar de lanzar excepciones.
    \item \texttt{event Action Rebuilt} Se lanza en el hilo principal justo después de (re)construir los datos del volumen. Permite refrescar cualquier elemento que dependa de la navegación.
    Los agentes vinculados lo emplean para volver a solicitar sus rutas de forma automática.
    \item \texttt{void Build()} Construye la estructura de navegación reemplazando los datos previos y disparando después el evento \texttt{Rebuilt}.
    Se emplea con el modo de construcción \textit{Manual} o para forzar una reconstrucción en tiempo de ejecución.
    \item \texttt{bool IsInsideVolume(Vector3 worldPos)} Devuelve \texttt{true} si el punto está dentro de la caja de contorno, con independencia de que la posición esté libre u ocupada.
    No requiere que el volumen esté construido.
    \item \texttt{bool IsNavigable(Vector3 worldPos)} Devuelve \texttt{true} si el punto está dentro del volumen y el vóxel que lo contiene está libre (no bloqueado por la geometría). Devuelve \texttt{false} si el volumen no está listo o el punto queda fuera u obstaculizado.
    \item \texttt{Vector3 ClampToVolume(Vector3 worldPos)} Devuelve el punto más cercano a \texttt{worldPos} que queda dentro de la caja de contorno del volumen.
    Si ya estaba dentro, lo devuelve sin cambios.
    \item \texttt{bool TrySnapToNavigable(Vector3 worldPos, float maxDistance, out Vector3 result)} Busca el centro del vóxel libre más cercano a \texttt{worldPos}, explorando hacia afuera en capas concéntricas sobre la cuadrícula de vóxeles más fina, hasta un máximo de \texttt{maxDistance} metros (el punto de consulta se ajusta primero al interior del volumen).
    Devuelve el punto hallado en \texttt{result} y retorna \texttt{true} si tiene éxito, o \texttt{false} si el volumen no está listo, \texttt{maxDistance} es negativo o no hay ningún vóxel libre dentro del rango.
    \item \texttt{Vector3 GetRandomPoint()} Devuelve un punto navegable aleatorio dentro del volumen, o \texttt{Vector3.zero} si el volumen no está listo o no se encuentra ninguno.
    \item \texttt{Vector3 GetRandomPointInBounds(Bounds bounds)} Como \texttt{GetRandomPoint}, pero restringido a la intersección entre \texttt{bounds} y el volumen.
    \item \texttt{Vector3 GetRandomPointInSphere(Vector3 center, float radius)} Como \texttt{GetRandomPoint}, pero restringido a la intersección entre la esfera indicada y el volumen.
    \item \texttt{bool TryGetRandomPoint(out Vector3 point, int maxAttempts = 30)} Trata de encontrar un punto navegable aleatorio dentro del volumen a tomando repetidamente una muestra completamente aleatoria hasta que sea válida o hasta llegar al máximo número de intentos.
    \item \texttt{bool TryGetRandomPointInBounds(Bounds bounds, out Vector3 point, int maxAttempts = 30)} Igual que \texttt{TryGetRandomPoint}, muestreando dentro de la intersección entre \texttt{bounds} y el volumen.
    \item \texttt{bool TryGetRandomPointInSphere(Vector3 center, float radius, out Vector3 point, int maxAttempts = 30)} Igual que \texttt{TryGetRandomPoint}, muestreando dentro de la intersección entre la esfera y el volumen.
\end{itemize}

\subsection{\texttt{NavVolumeAgent}}

\begin{itemize}
    \item \texttt{NavVolumeSpace NavVolumeSpace \{ get; \}} El volumen al que el agente está vinculado en ese momento, resuelto automáticamente al inicio.
    \item \texttt{void SetDestination(Vector3 goal)} Solicita una ruta desde la posición actual del agente hasta \texttt{goal} y empieza a volar en cuanto la encuentra.
    La búsqueda se ejecuta en segundo plano y, si se vuelve a llamar, cancela cualquier petición en curso.
    El destino se recuerda, de modo que el agente recalcula la ruta automáticamente si su volumen se reconstruye.
    El destino debería ser un punto navegable, en caso de duda, conviene ajustarlo antes con \texttt{TrySnapToNavigable}.
\end{itemize}
