\chapter{Manual de usuario}

Este apéndice describe el uso de la herramienta desde el editor de Unity, orientado tanto a perfiles técnicos como a diseñadores de niveles y artistas.
Para la instalación previa del paquete, consultar el Apéndice~\ref{anexo:instalacion}.

\section{Agentes}

El primer paso para el uso de la herramienta, es crear la configuración para un tipo de agente.
Para ello se pulsa el click derecho sobre la ventana de proyecto y se selecciona la opción de "Create > NavVolume > Agent Type", tal y como se ilustra en la Figura~\ref{fig:creacionagente}.

\begin{figure}[htp]
  \centering
  \includegraphics[width=0.75\textwidth]{imaxes/creacionagente.png}
  \caption[Creación de la configuración para un agente.]
  {Creación de la configuración para un agente.
  Imagen obtenida por medio de una captura de pantalla.}
  \label{fig:creacionagente}
\end{figure}

Una vez se haya creado este fichero de configuración, si hacemos click sobre este podremos modificar una serie de parámetros, mostrados en la Figura~\ref{fig:configuracionagente}:

\begin{itemize}
    \item \textbf{Radius:} radio del agente, empleado para evitar tanto la geometría del terreno como a otros agentes.
    \item \textbf{Cost mode:} si se elige la opción de \textbf{Node Count} el agente ponderará en la búsqueda todos los nodos por igual, lo que se traduce en resultados que tardan menos en calcularse pero que tienden a moverse hacia el espacio más abierto.
    En cambio, si se elige \textbf{Euclidean distance} los resultados serán más exactos a cambio de rendimiento.
    En espacios muy abiertos es recomendable usar el primer método ya que el postprocesado de caminos suele compensar el error de precisión que cometen.
    \item \textbf{Heuristic Weight:} peso que se le da a la heurística en la búsqueda de caminos.
    En consecuencia si el valor es 1 devolverá el camino óptimo siempre y si es mayor calculará la ruta más rápido sacrificando la optimialidad.
    \item \textbf{Avoidance Neighbor Range:} distancia máxima a la que otros agentes se consideran para la evasión.
    \item \textbf{Avoidance Max Neighbors:} número máximo de agentes que se consideran a la vez para la evasión 
    \item \textbf{Avoidance Time Horizon Agents:} cuántos segundos mira en el futuro para evitar colisiones con otros agentes.
    \item \textbf{Avoidance Time Horizon Obstacles:}cuántos segundos mira en el futuro para evitar colisiones con obstáculos dinámicos.
\end{itemize}

\begin{figure}[htp]
  \centering
  \includegraphics[width=0.75\textwidth]{imaxes/configuracionagente.png}
  \caption[Parámetros de configuración para un agente.]
  {Parámetros de configuración para un agente.
  Imagen obtenida por medio de una captura de pantalla.}
  \label{fig:configuracionagente}
\end{figure}

Para conseguir que un objeto se mueva, debemos añadirle el componente \texttt{NavVolumeAgent}, ilustrado en la Figura~\ref{fig:componenteagente}, y luego asignarle a este su configuración.
Desde el inspector podemos modificar los siguientes parámetros:

\begin{itemize}
    \item \textbf{Agent Type:} fichero de configuración del agente.
    \item \textbf{Speed:} velocidad a la que se moverá el agente.
    \item \textbf{Angular Speed:} velocidad a la que rotará el agente en $^\circ/\mathrm{s}$.
    \item \textbf{Is Avoidance Enabled:} checkbox para activar o desactivar la evasión de otros agentes u obstáculos.
    \item \textbf{Freeze Rotation:} por defecto un agente rota para estar orientado hacia donde se mueve.
    Al marcar estas casillas podemos limitar la rotación en cierto eje.
    Por ejemplo, si usamos un dron a modo de agente, queremos permitirle a este rotar solo alrededor del eje $Y$.
\end{itemize}
